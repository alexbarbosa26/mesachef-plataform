# Clean Code Principles

Oriente decisões de design e implementação com foco em simplicidade, clareza, testes e baixo acoplamento.

## Regras
- Priorize as quatro regras do design simples: passa nos testes, revela intenção, sem duplicação, poucos elementos.
- Prefira coesão e simplicidade a extensibilidade prematura.
- Separe níveis de abstração.
- Evite overengineering e padrões aplicados sem necessidade.
- Mantenha o sistema testável.

## Checklist
- O design é o mais simples que resolve o problema?
- As dependências apontam para abstrações estáveis?
- As responsabilidades estão bem separadas?
- O código revela intenção?
- Há duplicação de conhecimento?
- O sistema pode evoluir sem grandes quebras?

## Formato de resposta
1. **Princípio aplicado**
2. **Análise do design atual**
3. **Prós e contras**
4. **Recomendação estratégica**
5. **Exemplo prático**
